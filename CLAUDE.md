# worldmachines

> **Environment rules, keys & safety policies:** see [Code/CLAUDE.md](../CLAUDE.md) — read before starting work.

Collaborative project to develop psychohistorical models based on Venkatesh Rao's world machines theory. A static website on Cloudflare Pages that aggregates links to articles submitted by approved collaborators, with full-text extraction stored for future AI backend use.

Live at **worldmachines.org**.

## Stack

- Static HTML · CSS · JS (no build framework)
- Cloudflare Pages (hosting + Pages Functions for serverless endpoints)
- Cloudflare Zero Trust / Access (authentication gate)
- Cloudflare KV (handle registry with full profile)
- GitHub Actions (article ingestion pipeline)
- Python + `trafilatura` (full-text extraction)

```
  api/
    submit.js               ← Pages Function: handles form POST, fires GH dispatch
    profile.js              ← Pages Function: GET/POST contributor profile from KV
    contributors.js         ← Pages Function: returns all contributor profiles
    admin/
      handles.js            ← Pages Function: CRUD for KV handle registry

  ingest.py                 ← fetches URL, extracts text+date, writes content/articles/<slug>.json
  build.py                  ← reads content/articles/*.json → regenerates index/contributions/resources HTML
  backfill_dates.py         ← one-off: populated published_at for essays with URLs via trafilatura
  backfill_book_dates.py    ← one-off: populated published_at for books via Open Library API

  articles/                 ← one JSON file per submitted article (60 files as of 2026-05-04)
.github/
  workflows/
    ingest.yml              ← triggered by repository_dispatch, runs ingest + build + deploy
requirements.txt            ← trafilatura (used by GitHub Actions)
```

## Article JSON schema

Each file in `content/articles/` contains:
```json
{
  "slug": "the-modernity-machine-20260504-abc123",
  "url": "https://...",
  "title": "Extracted or fallback title",
  "handle": "vgr",
  "submitted_at": "2026-05-04T18:00:00Z",
  "type": "contribution | resource",
  "format": "essay | short story | paper | book",
  "published_at": "2025-01-26",
  "description": "Optional submitter note",
  "extraction_success": true,
  "extracted_text": "Full article text (or null if extraction failed)",
  "author": "Author Name (books/resources only)",
  "section": "Optional section/grouping label"
}
```

**type vs format:**
- `type` = `contribution` (authored by a project member) or `resource` (third-party). Drives badge color and which page it appears on. Resources are never fetched/ingested.
- `format` = `essay`, `short story`, `paper`, `book`. Drives the badge label.
- `published_at` = ISO date string for essays/contributions; 4-digit year string for books (e.g. `"1759"`). Falls back to `submitted_at` for display.

## Submission pipeline

1. Collaborator visits `worldmachines.org/submit` → Cloudflare Access email OTP gate
2. Form POSTs to `/api/submit` (Pages Function)
3. Function looks up submitter's handle+name from KV, fires `repository_dispatch` to GitHub
4. GitHub Actions: runs `ingest.py` (fetch + extract) → `build.py` (rebuild pages) → commits → pushes
5. Same GH Actions job runs `wrangler pages deploy` to push live

Resources (type=resource) skip fetching — only title+metadata stored.
Full text is stored in the JSON but not yet surfaced on the site — reserved for the future AI backend layer.

## Handle registry (KV)

KV namespace: `handles` (ID: `ca021379904746528348029c242faaff`)

Maps `email → JSON string` with fields: `handle`, `name`, `url`, `bio`.

Example value:
```json
{"handle": "vgr", "name": "Venkatesh Rao", "url": "https://venkateshrao.com", "bio": "..."}
```

Managed at `worldmachines.org/admin/handles`. To add a collaborator via CLI:
```bash
wrangler kv key put "email@example.com" '{"handle":"foo","name":"Full Name","url":"","bio":""}' \
  --namespace-id ca021379904746528348029c242faaff --remote
```
Then also add their email to the `/submit` Access policy in Cloudflare Zero Trust.
