# Frames and Framing

## Definition

A **frame** is an explanatory structure that defines entities by describing their relationship to other entities. Klein et al. (2007) define it as "a structure for accounting for the data and guiding the search for more data. It reflects a person's compiled experiences."

Frames can take multiple structural forms:
- **Story** — explaining chronology of events and causal relationships between them
- **Map** — explaining position by showing distances and directions to landmarks
- **Script** — explaining one's role as complementary to others' roles
- **Plan** — a sequence of intended actions
- **Mental model** — causal understanding of how a system works

## The Data-Frame Relationship

The core insight from [[Data-Frame Theory of Sensemaking]]: *neither data nor frame comes first.* The relationship is constitutive and bidirectional:

> "The data identify the relevant frame, and the frame determines which data are noticed. Neither of these comes first."

This is analogous to a hidden-figures task: once the frame becomes clear, you cannot *not* see the data it organizes. Before the frame, those same data are invisible.

Frames "accept" information as it becomes available, and the frame is changed by that information; it is used to "direct" exploratory activities that make more data available, by which it is further modified. This is Neisser's (1976) perceptual cycle applied to macrocognition.

## Intellectual Genealogy

Klein et al. synthesize a cluster of related prior concepts under the single term "frame":

| Theorist | Term | Key Feature |
|----------|------|-------------|
| Goffman (1974) | Frame | Culturally relative system of rules guiding behavior |
| Minsky (1975) | Frame | Data structure with slots and default values ("slot-filling") |
| Schank & Abelson (1977) | Script | Regularly occurring event sequence as template |
| Bartlett (1932) | Schema | Mental representation of event description structure |
| Neisser (1976) | Schema | Domain-specific structure directing external information search |
| Piaget (1952, 1954) | Schema | Mental representation of persistent object attributes |

All of these describe top-down cognitive structures that filter and organize experience. Klein et al. use "frame" as synthesis of all of them.

## What a Frame Is Not

A frame is *not* a collection of inferences drawn from the data, although it can include inferences. Paraphrasing Neisser: a frame is "that portion of the perceptual cycle that is internal to the perceiver, modifiable by experience, and specific to what is being perceived."

## Frame vs. Situation Awareness

Endsley's (1995) [[Situation Awareness]] describes sensemaking as a *state of knowledge*. The data-frame view differs critically: frames determine what counts as data in the first place. We are not just retrieving and assembling accurate information; we are constructing what "data" means. The frame is the constructive apparatus, not a filing cabinet being filled.

## Frame Selection and Anchors

Frames are selected from a small number of key anchors — typically 3–4 salient data elements. The initial anchor(s) elicit the initial frame, which then guides further data search. See [[Anchoring in Sensemaking]].

## Frame Preservation and Fixation

Because frames filter what gets noticed, inconsistent data can be explained away rather than treated as evidence against the frame. This is adaptive (frames need stability to function) and pathological (see [[Fixation Error]]). Feltovich, Coulson & Spiro (2001) catalog "knowledge shields" — tactics experts use to preserve frames against countervailing evidence: arguing from authority, invoking special cases, ignoring secondary effects.

## Piaget's Parallel

Elaborating a frame resembles Piaget's *assimilation* — fitting new data into existing structure. Reframing resembles *accommodation* — restructuring the frame to fit data that cannot be assimilated.
