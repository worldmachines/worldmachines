# Documentation as Empathy

## Core Idea

Good documentation is not a technical deliverable — it is an exercise in **deep empathy** with a future reader whose context, level, and specific problem are unknown. From [[Divine Documentation]]:

> "Writing good documentation is hard. It is an exercise in deep empathy with my user. Reaching into the future to give them all they need is part of creating good technology."

## The Time Travel Frame

Documentation is a form of **temporal empathy**: the writer projects into a future moment, inhabits a future reader's confusion, and leaves behind exactly what that reader will need. This is structurally similar to writing a will, or leaving instructions for your future self.

**The self-service compact**: "Often the future user is me and I like it when past me is nice to now me." This collapses the empathy target to the self across time — a lower-cost version of the same skill.

## The Two Modalities Contrasted

| Modality | Character | Metaphor |
|---|---|---|
| Socratic interlocutor | Active resistance, forces articulation | "Weight training" |
| Good documentation | Anticipatory care, removes friction | "A kindly spirit ancestor parting the mist" |

Both serve learning, but in opposite ways. The [[Socratic Method]] builds capability through struggle; documentation eliminates unnecessary struggle so real work can proceed.

## Why Good Documentation Is Rare

Documentation requires the author to:
1. Forget their expert context
2. Model a naive reader's mental model
3. Anticipate failure modes they've already transcended
4. Find this worth doing

All four are cognitively and motivationally expensive. Most documentation fails at step 1.
